# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Overview

This is an educational content repository about AI fluency, organized as structured Markdown documents with supporting images. There are no build tools, test suites, or programming frameworks — the primary work here is authoring and editing Markdown content.

## Content Structure

Each top-level directory corresponds to a module in a sequential curriculum:

| Directory | Topic |
|---|---|
| `01-what-is-a-machine/` | Machines, computers, AI history, and foundations |
| `02-ai-in-everyday-life/` | Real-world AI use cases |
| `03-ai-education-and-in-philippines/` | AI in education, Philippine context, and impact |
| `04-ai-fluency/` | AI fluency framework and Google Prompting Essentials |
| `05-generative-ai/` | Generative AI concepts and chatbots |
| `06-five-step-prompting-framework/` | Five-step prompting framework |

Each directory contains `.md` files (content) and `.png` images referenced inline using relative paths (e.g., `![alt](image.png)`).

## Content Conventions

- Images are embedded using Markdown syntax with relative paths local to each module directory.
- File numbering within directories follows the module number (e.g., `03.0-`, `03.1-`, `03.2-` inside `03-ai-education-and-in-philippines/`).
- Headings use `#` through `###` hierarchy; key definitions are bolded with `**`.
- Tables are used for structured comparisons (frameworks, module breakdowns, etc.).
